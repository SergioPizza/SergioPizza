## *Изменение 2*

# H1 Сценарий регистрации нового пользователя


Пользователь должен иметь возможность зарегистрироваться на сайте заказа пиццы для создания учетной записи и пользования другими функциями сайта.


## Акторы:


**Пользователь:** человек, который хочет зарегистрироваться на сайте заказа пиццы.


**Система:** веб-сайт заказа пиццы.


## Предусловия:


Пользователь открыл сайт заказа пиццы.


## Основной поток:


1. Пользователь нажимает кнопку "Регистрация".

2. Система отображает форму для ввода данных для регистрации (имя, фамилия, номер телефона, адрес электронной почты, пароль).

3. Пользователь вводит свои данные в форму.

4. Система проверяет введенные данные на корректность и уникальность.

Если данные введены правильно и пользователь согласен с правилами сайта, система создает учетную запись для пользователя и автоматически авторизует его на сайте.

5. Система перенаправляет пользователя на страницу заказа пиццы.


## Альтернативный поток:


Если пользователь уже имеет учетную запись, он может нажать на ссылку "Войти", после чего система перенаправит его на страницу авторизации.


## Исключения:


*На шаге 4:* Если данные введены некорректно, то система уведомляет пользователя о невозможности создания учетной записи.


## Постусловия:


Пользователь успешно зарегистрирован на сайте заказа пиццы и может использовать свою учетную запись для оформления заказов и других функций сайта.
